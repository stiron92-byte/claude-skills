# Content Repurpose - 원소스 멀티콘텐츠 자동 생성 스킬

콘텐츠 대량 생산 스킬: 하나의 원본 소스(유튜브 대본, 강의 텍스트, 회의록 등)를 여러 플랫폼에 최적화된 콘텐츠로 자동 변환합니다.

Use proactively when user wants to repurpose content, convert video scripts to multi-platform posts, or analyze content gaps.

Triggers: repurpose, 리퍼포징, 콘텐츠 변환, 멀티 콘텐츠, 원소스, 콘텐츠 생산, 콘텐츠 감사, 갭 분석, content audit, content gap,
블로그 변환, SNS 변환, 뉴스레터 변환, 대본 변환, 스크립트 변환, 콘텐츠 재활용

---

## 실행 모드

이 스킬은 3가지 모드로 동작합니다. 사용자가 모드를 명시하지 않으면 **변환 모드**를 기본으로 실행합니다.

### 모드 1: 변환 (기본)
원본 콘텐츠를 선택한 플랫폼에 맞게 변환합니다.

### 모드 2: 감사 (audit)
기존 콘텐츠 라이브러리를 분석하여 부족한 주제, 중복 콘텐츠, 개선 포인트를 도출합니다.

### 모드 3: 갭 분석 (gap)
경쟁 채널/사이트 대비 콘텐츠 갭을 분석합니다.

---

## 모드 1: 콘텐츠 변환 프로세스

### 1단계: 원본 소스 확인

사용자에게 다음을 확인합니다:

1. **원본 콘텐츠**: 유튜브 대본, 강의 텍스트, 회의록, 블로그 글 등
   - 사용자가 직접 텍스트를 붙여넣거나 파일 경로를 제공
   - `$ARGUMENTS`에 텍스트가 포함되어 있으면 그것을 원본으로 사용

2. **콘텐츠 분야/톤**: 사용자가 지정하지 않으면 질문으로 파악
   - 예: 교육, 테크, 뷰티, 비즈니스, 라이프스타일, 금융 등
   - 톤: 전문적, 캐주얼, 친근한, 유머러스 등

### 2단계: 플랫폼 선택

사용자에게 **생성할 플랫폼을 선택**하도록 안내합니다.
AskUserQuestion 도구를 사용하여 multiSelect로 질문합니다:

**선택 가능한 플랫폼:**
- SEO 블로그 포스트
- X (Twitter) 스레드
- LinkedIn 게시물
- Instagram 캡션
- 뉴스레터 본문
- 썸네일/비주얼 가이드
- YouTube Shorts 스크립트
- TikTok 스크립트
- Facebook 게시물
- Pinterest 핀 설명
- Threads 게시물
- 브런치 글

### 3단계: 플랫폼별 콘텐츠 생성

각 플랫폼에 대해 아래 규칙을 적용하여 콘텐츠를 생성합니다.
**모든 출력은 한국어로 작성합니다.**

---

#### SEO 블로그 포스트
```
[형식]
- 제목: SEO 키워드 포함, 50자 이내
- 메타 디스크립션: 155자 이내
- 본문: 1,500~2,500자
- 구조: H2/H3 소제목, 번호 리스트, 핵심 요약 박스
- SEO: 주요 키워드 3~5개 자연 삽입, 내부 링크 자리 표시
- CTA: 글 하단에 행동 유도 문구

[톤] 정보 전달형, 신뢰감 있는 문체
[필수 요소] 도입부 훅 → 본론 (소제목별 정리) → 핵심 요약 → CTA
```

#### X (Twitter) 스레드
```
[형식]
- 1/N 형식의 스레드
- 각 트윗: 280자(한글 기준 140자) 이내
- 첫 트윗: 강력한 훅 (질문, 충격적 사실, 반전)
- 마지막 트윗: CTA + 리트윗 유도
- 총 5~10개 트윗

[톤] 간결, 임팩트, 구어체
[필수 요소] 훅 → 핵심 포인트 → 인사이트 → CTA
[해시태그] 마지막 트윗에 2~3개
```

#### LinkedIn 게시물
```
[형식]
- 본문: 1,300자 이내 (약 200~250단어)
- 첫 3줄: 강력한 훅 (접히기 전에 보이는 영역)
- 단락: 1~2문장씩 짧게
- 줄바꿈 적극 활용

[톤] 전문가적, 인사이트 중심, 경험 공유형
[필수 요소] 훅 → 경험/스토리 → 인사이트 → 질문형 CTA
[해시태그] 본문 하단 3~5개
```

#### Instagram 캡션
```
[형식]
- 캡션: 2,200자 이내
- 첫 줄: 강력한 훅 (피드에서 보이는 영역)
- 줄바꿈과 이모지로 가독성 확보
- CTA: 저장/공유/댓글 유도

[톤] 친근, 감성적, 공감형
[필수 요소] 훅 → 본문 → CTA → 해시태그
[해시태그] 20~30개 (관련성 높은 것 우선)
[비주얼 제안] 어울리는 이미지/캐러셀 구성 제안
```

#### 뉴스레터 본문
```
[형식]
- 제목(subject line): 50자 이내, 오픈율 높이는 형식
- 프리헤더: 80자 이내
- 본문: 500~800단어
- 구조: 인사 → 핵심 내용 → 추가 자료 → CTA

[톤] 1:1 대화형, 친밀하면서도 가치 전달
[필수 요소] 개인적 인사 → 핵심 가치 → 부가 정보 → CTA 버튼 텍스트
```

#### 썸네일/비주얼 가이드
```
[형식]
- 썸네일 텍스트: 핵심 문구 (4~8자)
- 색상 제안: 주 색상 + 강조 색상
- 레이아웃: 텍스트 위치, 이미지 구성 설명
- 표정/제스처 제안 (인물 포함 시)
- 대체 시안 2~3개

[참고] 16:9 비율 기준, 모바일에서도 가독성 확보
```

#### YouTube Shorts / TikTok 스크립트
```
[형식]
- 길이: 30~60초 분량 (150~300자)
- 구조: 훅(3초) → 본론 → 반전/CTA
- 첫 문장: "~하는 방법", "~하지 마세요", 질문형 등 강력한 훅

[톤] 빠르고 에너지 넘치는 구어체
[필수 요소] 3초 훅 → 핵심 정보 → 팔로우/구독 CTA
[자막 텍스트] 화면에 표시할 핵심 텍스트 별도 제공
```

#### Facebook 게시물
```
[형식]
- 본문: 500자 이내
- 이미지/링크 설명 포함
- 공유를 유도하는 구성

[톤] 커뮤니티형, 대화 유도
[필수 요소] 훅 → 본문 → 의견 요청형 CTA
```

#### Pinterest 핀 설명
```
[형식]
- 제목: 100자 이내, 키워드 포함
- 설명: 500자 이내
- 보드 제안

[톤] 실용적, 검색 최적화
[필수 요소] 키워드 풍부한 설명 + 행동 유도
```

#### Threads 게시물
```
[형식]
- 500자 이내
- 대화형 구성
- 이미지 없이도 임팩트 있는 텍스트

[톤] 캐주얼, 트렌디, 밈 감성 가능
[필수 요소] 훅 → 짧은 본문 → 반응 유도
```

#### 브런치 글
```
[형식]
- 제목: 감성적이면서 검색 가능한 형태
- 본문: 2,000~4,000자
- 소제목 활용, 문단 나누기
- 인용구 활용

[톤] 에세이형, 깊이 있는 서술
[필수 요소] 감성적 도입 → 본론 → 여운 있는 마무리
```

### 4단계: 출력

각 플랫폼 콘텐츠를 명확히 구분하여 출력합니다:

```
## [플랫폼명] 콘텐츠

[생성된 콘텐츠]

---
```

마지막에 전체 요약 표를 제공합니다:

```
| 플랫폼 | 글자 수 | 핵심 키워드 | CTA |
|--------|---------|------------|-----|
| ...    | ...     | ...        | ... |
```

## 모드 2: 콘텐츠 감사 (audit)

사용자에게 다음 정보를 수집합니다:

1. **분석 대상**: 콘텐츠 목록 (제목, URL, 주제, 날짜 등)
   - 텍스트로 직접 입력 또는 파일 경로 제공
2. **분석 기준**: 주제 커버리지, 성과 지표, 업데이트 필요성

분석 결과를 다음 형식으로 출력합니다:

```
## 콘텐츠 감사 결과

### 1. 주제별 분포
| 주제 카테고리 | 콘텐츠 수 | 비율 | 평가 |
|-------------|----------|------|------|

### 2. 부족한 주제 영역
- [영역]: [이유]

### 3. 업데이트 필요 콘텐츠
- [제목]: [이유]

### 4. 중복/통합 가능 콘텐츠
- [제목 A] + [제목 B]: [통합 제안]

### 5. 권장 액션
1. [우선순위 1 액션]
2. [우선순위 2 액션]
...
```

---

## 모드 3: 갭 분석 (gap)

사용자에게 다음 정보를 수집합니다:

1. **내 콘텐츠**: 현재 보유한 콘텐츠 목록/주제
2. **경쟁 대상**: 경쟁 채널/사이트 정보
3. **분석 범위**: 주제, 형식, 플랫폼 등

분석 결과를 다음 형식으로 출력합니다:

```
## 콘텐츠 갭 분석 결과

### 1. 경쟁 대비 미보유 주제
| 주제 | 경쟁사 보유 | 우선순위 | 난이도 |
|------|-----------|---------|-------|

### 2. 형식 갭
- 경쟁사는 [형식]을 활용하지만 우리는 미보유

### 3. 플랫폼 갭
- 경쟁사가 활동하지만 우리가 부재한 플랫폼

### 4. 콘텐츠 생성 로드맵
| 순서 | 주제 | 추천 플랫폼 | 기대 효과 |
|------|------|-----------|----------|
```

---

## 대화 흐름 규칙

1. **정보가 부족하면 반드시 질문**: 원본 콘텐츠, 분야, 톤 등 필수 정보가 없으면 생성하지 말고 질문합니다.
2. **한국어 전용**: 모든 출력은 한국어로 작성합니다.
3. **플랫폼 선택은 사용자에게 위임**: 임의로 모든 플랫폼을 생성하지 않습니다.
4. **원본 충실도**: 원본의 핵심 메시지와 논점을 왜곡하지 않습니다.
5. **플랫폼 특성 준수**: 각 플랫폼의 글자 수 제한, 톤, 형식을 반드시 따릅니다.
6. **파일 저장 제안**: 생성된 콘텐츠가 많을 경우, 마크다운 파일로 저장할지 사용자에게 제안합니다.

---

## 인자 처리

`$ARGUMENTS` 값에 따라 모드를 자동 판별합니다:

- `audit` 또는 `감사`로 시작 → 모드 2
- `gap` 또는 `갭` 또는 `갭분석`으로 시작 → 모드 3
- 그 외 → 모드 1 (변환)
- 빈 값 → 모드 1, 원본 콘텐츠 입력을 요청
