# 실습 3. 콘텐츠 리서치 자동화 — "트렌드 찾기를 자동으로"

> 소요 시간: 1시간 | 난이도: ★☆☆
> 바로 쓸 수 있는 곳: 유튜브 주제 선정, 블로그 기획, 뉴스레터 발행

유튜브 채널을 운영하든, 블로그를 쓰든, 뉴스레터를 발행하든 결국 가장 먼저 해야 하는 일은 같습니다. **"이번 주에는 뭘 다뤄야 하지?"** 이 질문에 답하기 위해 여러 뉴스 사이트를 돌아다니고, 트렌드를 검색하고, 아이디어를 정리하는 데 매주 2~3시간을 쓰고 계시진 않나요?

이번 실습에서는 이 과정을 통째로 자동화하는 Claude Code 스킬을 직접 만들고, 설치하고, 사용해봅니다. 한 번 설치하면 그 다음부터는 이렇게만 말하면 됩니다.

```
유튜브 컨텐츠 뭐 만들지 찾아봐줘
```

Claude가 알아서 뉴스를 수집하고, 분석하고, 여러분의 채널에 맞는 영상 주제 5개를 뽑아줍니다.

---

## 이번 실습에서 만드는 것

```
관심 분야 뉴스 자동 수집 → AI 분석 → 콘텐츠 아이디어 리스트 + 요약 브리핑
```

완성하면 이런 결과물을 받게 됩니다:

**유튜브용이라면:**
- 영상 제목 5개 (클릭을 유도하는 제목)
- 각 주제별 썸네일 아이디어
- 타겟 시청자 분석
- 예상 관심도 (높음/중간/낮음)

**블로그용이라면:**
- SEO 최적화 제목 5개
- 핵심 키워드 3~5개
- 글 구조 (소제목 목록)

**뉴스레터용이라면:**
- 이번 주 TOP 3 뉴스 요약
- 심화 분석 1개
- 추천 링크와 마무리 멘트

---

## 3.1 완성된 스킬 설치하기

이 실습에는 미리 만들어둔 스킬 패키지가 포함되어 있습니다. 먼저 설치부터 해보겠습니다.

### 스킬 파일 복사

```bash
cp -r content-research-skill ~/.claude/skills/content-research
```

이 한 줄이면 설치 끝입니다. `~/.claude/skills/` 는 Claude Code가 스킬을 찾는 개인 디렉토리입니다. 여기에 넣으면 어떤 프로젝트에서든 사용할 수 있습니다.

### 설치 확인

Claude Code를 열고 `/` 를 입력해보세요. 자동완성 목록에 `content-research`가 보이면 성공입니다.

---

## 3.2 스킬 사용해보기

설치했으니 바로 써봅시다. Claude Code에 이렇게 말해보세요:

```
유튜브 컨텐츠 제작하려고 하는데 어떤 컨텐츠가 좋을지 찾아봐주세요
```

그러면 Claude가 바로 작업에 들어가는 게 아니라, **먼저 질문을 합니다:**

```
콘텐츠 리서치를 시작하겠습니다. 먼저 몇 가지 확인이 필요합니다.

1. 어떤 분야의 트렌드를 수집할까요? (예: 테크/AI, 요리, 금융, 여행, 한국IT)
2. 수집할 RSS 피드 URL이 있으신가요? 없으면 분야에 맞게 추천해드립니다.
3. 콘텐츠 용도가 무엇인가요? (유튜브 주제 선정 / 블로그 기획 / 뉴스레터 / 일반)
4. 브리핑을 한국어로 작성할까요, 영어로 작성할까요?
```

이렇게 답하면 됩니다:

```
1. 테크/AI
2. 추천해주세요
3. 유튜브 주제 선정
4. 한국어
```

Claude가 답변을 받은 뒤에야 설정 파일을 만들고, 뉴스를 수집하고, 분석을 시작합니다. **왜 먼저 묻고 나서 작업할까요?** 이것이 바로 스킬의 힘입니다. SKILL.md 안에 "질문을 먼저 하라"고 적어두었기 때문입니다.

> **왜 질문 먼저?**
> 스킬 없이 같은 요청을 하면 Claude는 "알아서" 추측해서 코드를 만듭니다. 결과가 마음에 안 들면 다시 만들어야 하죠. 스킬은 이런 시행착오를 없앱니다. 처음부터 필요한 정보를 모은 뒤 한 번에 정확하게 실행합니다.

### 슬래시 명령어로도 호출 가능

자연어 대신 직접 호출할 수도 있습니다:

```
/content-research
/content-research tech
```

인자를 함께 전달하면 해당 질문은 건너뜁니다.

---

## 3.3 스킬 안에서 일어나는 일

뒤에서 어떤 일이 벌어지는지 살펴봅시다.

### Step 1: RSS로 뉴스 수집

RSS는 웹사이트가 보내주는 "새 글 알림"입니다. 대부분의 뉴스 사이트, 블로그, 미디어가 RSS를 제공합니다. 스킬은 `feedparser`라는 Python 라이브러리로 여러 RSS 피드에서 최신 기사를 자동 수집합니다.

```python
# rss_collector.py 핵심 로직
import feedparser

feed = feedparser.parse("https://techcrunch.com/feed/")
for entry in feed.entries[:10]:
    print(entry.title)    # 기사 제목
    print(entry.summary)  # 기사 요약
    print(entry.link)     # 원문 링크
```

이 몇 줄로 TechCrunch의 최신 기사 10개를 가져올 수 있습니다.

### Step 2: Claude API로 분석

수집된 기사를 Claude API에 보내 분석합니다. 용도에 따라 프롬프트가 달라집니다:

| 용도 | Claude가 하는 일 |
|------|-----------------|
| 유튜브 | 영상 제목 5개 + 썸네일 아이디어 + 시청자 분석 |
| 블로그 | SEO 제목 + 키워드 + 글 구조 |
| 뉴스레터 | TOP 3 뉴스 요약 + 심화 분석 + 마무리 |

### Step 3: 결과 출력

분석 결과는 마크다운 파일로 저장됩니다:

```
output/
├── articles_20260226_090000.json   ← 수집된 기사 원본
└── briefing_20260226_090000.md     ← 분석 브리핑
```

## 3.4 스킬의 구조 이해하기

이제 이 스킬이 어떻게 만들어져 있는지 살펴봅시다. 향후 여러분만의 스킬을 만들 때 동일한 구조를 따르면 됩니다.

### 디렉토리 구조

```
content-research/
├── SKILL.md                 ← 핵심: Claude가 읽는 지시서
├── scripts/
│   ├── main.py              ← 메인 파이프라인
│   ├── rss_collector.py     ← RSS 수집
│   ├── content_analyzer.py  ← Claude API 분석
│   ├── setup_wizard.py      ← 대화형 설정
│   └── requirements.txt     ← Python 의존성
├── templates/
│   └── config.example.yaml  ← 설정 템플릿
└── references/
    └── SETUP-GUIDE.md       ← 상세 가이드
```

이 중에서 가장 중요한 파일은 단연 `SKILL.md`입니다.

### SKILL.md — 스킬의 두뇌

SKILL.md는 크게 두 부분으로 나뉩니다:

**1부: YAML 헤더 (Claude가 스킬을 찾는 데 사용)**

```yaml
---
name: content-research
description: Use when the user wants content ideas, topic suggestions,
  trend research, or news collection. 트렌드 찾기, 콘텐츠 리서치,
  유튜브 주제 선정, 유튜브 컨텐츠 뭐 만들지, 어떤 컨텐츠가 좋을지,
  블로그 기획, 뉴스레터 발행, 콘텐츠 아이디어, 콘텐츠 캘린더...
allowed-tools: Bash(python *) Bash(pip *)
---
```

**2부: 마크다운 본문 (Claude가 실행할 때 따르는 지시)**

```markdown
## ⛔ 절대 규칙
코드 생성, 파일 생성, 스크립트 실행을 하기 전에
반드시 아래 질문을 모두 완료하라.

## Step 1: 질문 (반드시 먼저 실행)
## Step 2: config 생성 (답변 확보 후)
## Step 3: 파이프라인 실행
```

### description — 스킬이 연결되는 비밀

여기서 한 가지 중요한 점을 짚고 넘어가겠습니다. Claude Code는 스킬을 어떻게 찾을까요?

```
사용자 발화 → description 키워드 매칭 → SKILL.md 본문 로드 → 실행
```

Claude Code가 시작되면 설치된 모든 스킬의 `name`과 `description`을 메모리에 올립니다. 사용자가 무언가를 말하면, 이 description들과 대조하여 가장 적합한 스킬을 찾습니다.

그래서 description에는 사용자가 **실제로 말할 법한 표현**을 넣어야 합니다:

```yaml
# ❌ 이렇게 쓰면 "유튜브 컨텐츠 뭐 만들지"로는 연결 안 됨
description: RSS feed collector and content analyzer

# ✅ 이렇게 써야 자연어 발화에 연결됨
description: 유튜브 주제 선정, 유튜브 컨텐츠 뭐 만들지,
  어떤 컨텐츠가 좋을지, 블로그 글감, 트렌드 찾기...
```

**핵심 원칙:**
- 기술 용어보다 **일상 표현**을 넣으세요
- **한국어와 영어** 모두 포함하세요
- 최대 1,024자까지 쓸 수 있으니 키워드를 다양하게 넣으세요

## 3.5 내 분야에 맞게 커스텀하기

이 스킬의 진짜 강점은 데이터 소스만 바꾸면 어떤 분야든 적용된다는 점입니다. Claude에게 질문받을 때 분야와 용도만 바꿔서 답하면 됩니다.

| 분야 | 이렇게 답하면 | 이런 결과가 나옵니다 |
|------|-------------|-------------------|
| 요리 유튜버 | 분야: 요리, 용도: 유튜브 | "이번 주 떠오르는 레시피" 영상 주제 5개 |
| 테크 유튜버 | 분야: 테크/AI, 용도: 유튜브 | "이번 주 주목할 제품/기술" 영상 주제 5개 |
| 금융 블로거 | 분야: 금융, 용도: 블로그 | "오늘의 시장 동향" SEO 블로그 글 구조 |
| 여행 뉴스레터 | 분야: 여행, 용도: 뉴스레터 | "이번 달 핫한 여행지" 뉴스레터 초안 |

더 세밀한 조정이 필요하다면 config.yaml의 `analysis_prompt`에 커스텀 프롬프트를 직접 작성할 수도 있습니다:

```yaml
analysis_prompt: "이 뉴스에서 30대 직장인 타겟 유튜브 주제 5개를 뽑아줘.
  각 주제마다 예상 조회수와 경쟁도를 분석해."
```

## 3.6 실습: 직접 해보기

지금 바로 따라해보세요. 15분이면 충분합니다.

### Step 1: 스킬 설치 (1분)

```bash
cp -r content-research-skill ~/.claude/skills/content-research
```

### Step 2: Claude Code에서 호출 (1분)

Claude Code를 열고 이렇게 말해보세요:

```
요즘 뜨는 트렌드 알려줘
```

### Step 3: 질문에 답하기 (2분)

Claude의 4가지 질문에 답합니다. RSS를 모르면 "추천해주세요"라고 하면 됩니다.

### Step 4: 결과 확인 (5분)

`.env` 파일에 Anthropic API 키를 입력하면 파이프라인이 실행되고, `output/` 폴더에 브리핑이 생성됩니다.

### Step 5: 3개월 캘린더 (선택, 5분)

브리핑이 마음에 들면 이렇게 말해보세요:

```
이 결과로 3개월 콘텐츠 캘린더도 만들어줘
```

주 2회 발행 기준, 24개 콘텐츠의 일정표가 자동으로 만들어집니다.

## 3.7 여기서 배운 것

이번 실습을 통해 우리는 다음을 배웠습니다:

| 개념 | 배운 것 |
|------|--------|
| **스킬 설치** | `~/.claude/skills/`에 디렉토리를 복사하면 끝 |
| **SKILL.md 구조** | YAML 헤더(매칭용) + 마크다운 본문(실행 지시) |
| **description의 역할** | 사용자 발화와 스킬을 연결하는 유일한 통로 |
| **질문 우선 패턴** | "절대 규칙"으로 Claude의 행동 순서를 제어 |
| **RSS 수집** | feedparser로 웹사이트의 새 글을 자동 수집 |
| **파이프라인** | 수집 → 분석 → 출력의 자동화 흐름 |

다음 실습에서는 이 패턴을 확장하여 여러분만의 커스텀 스킬을 처음부터 만들어봅니다.
