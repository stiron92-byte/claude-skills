# 실전 프로젝트 — 문서·PPT 자동화: "주간 보고를 15분에 끝내기"

---

## 이런 분에게 추천합니다

매주 월요일 아침, 지난주 실적 데이터를 열어 엑셀에서 숫자를 정리하고, 파워포인트를 열어 차트를 붙여넣고, 이메일까지 써서 보내는 데 1~2시간을 쓰고 계신가요?

이번 프로젝트에서는 **파일 하나만 던지면 보고용 PPT 5장과 요약 이메일이 자동으로 완성**되는 Claude 스킬을 직접 만들어봅니다.

| 항목 | 내용 |
|------|------|
| 만드는 것 | 파일 → 주간 보고 PPT + 요약 이메일 자동 생성 스킬 |
| 소요 시간 | 약 1시간 |
| 난이도 | ★☆☆ |
| 바로 쓸 수 있는 곳 | 주간/월간 보고, 팀 미팅 자료, 제안서, 포트폴리오 |

## 완성하면 이렇게 됩니다

엑셀 파일 하나를 Claude에 전달하면서 "이 데이터로 주간 보고 만들어줘"라고 말하면, 약 30초 뒤 다음 결과물이 자동으로 생성됩니다.

```
output/
├── weekly_report.pptx     ← PPT 5장 (표지, KPI, 차트, 이슈, 계획)
├── charts/                ← 라인·파이·막대 차트 이미지
├── email_executive.md     ← 임원용 요약 이메일
├── email_team.md          ← 팀원용 상세 이메일
└── analysis.json          ← 분석 원본 데이터
```

PPT에는 일별 매출 추이 차트, 카테고리별 비중 파이 차트, TOP 5 상품 순위까지 자동으로 들어갑니다. 이메일은 수신자에 따라 톤이 달라집니다. 임원에게는 핵심 숫자 3줄, 팀원에게는 액션 아이템이 포함된 상세 버전이 나옵니다.

## 핵심 개념: Claude 스킬이란?

본격적으로 만들기에 앞서, 이 프로젝트의 핵심 개념인 "스킬"을 짚고 넘어가겠습니다.

Claude 스킬은 **특정 작업에 대한 전문 지식을 Claude에게 가르치는 방법**입니다. 일반적인 대화에서 Claude는 범용적으로 답변하지만, 스킬을 등록하면 "이 상황에서는 이렇게 처리해라"라는 구체적인 가이드를 따르게 됩니다.

스킬의 핵심은 `SKILL.md` 파일입니다. 이 파일 하나가 곧 스킬의 정체성이며, 두 부분으로 구성됩니다.

```markdown
---
name: doc-automation
description: 다양한 형식의 파일을 분석하여 주간 보고 PPT와
             요약 이메일을 자동 생성합니다.
---

# 여기서부터 스킬의 상세 지시사항을 작성합니다
```

상단의 YAML 프론트매터(`---`로 감싼 부분)에 이름과 설명을 적고, 그 아래에 Claude가 따라야 할 워크플로우, 지원 파일 형식, 실행 명령어 등을 마크다운으로 작성합니다.

여기서 `description` 필드가 특히 중요합니다. **Claude는 이 설명을 읽고 사용자의 요청이 이 스킬에 해당하는지를 판단**합니다. 예를 들어 사용자가 "이 엑셀로 보고서 만들어줘"라고 말하면, Claude는 등록된 스킬들의 description을 훑어보고 가장 적합한 스킬을 활성화합니다.

> **핵심 포인트**: 스킬은 파일 확장자나 특정 명령어가 아니라, **사용자의 발화(대화 내용)**를 기준으로 연결됩니다.

## 스킬이 작동하는 흐름

이 스킬의 내부 파이프라인은 3단계로 구성됩니다.

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  Step 1      │    │  Step 2      │    │  Step 3      │
│  파일 분석    │ →  │  PPT 생성    │ →  │  이메일 생성  │
│              │    │              │    │              │
│ CSV → 차트   │    │ 5장 슬라이드  │    │ 임원용/팀원용 │
│ PDF → 텍스트  │    │ 브랜드 적용   │    │ 톤 자동 변환  │
└──────────────┘    └──────────────┘    └──────────────┘
```

**Step 1**에서는 입력 파일의 확장자를 자동으로 감지합니다. 엑셀이면 숫자를 분석하고 차트를 그리고, PDF나 Word면 텍스트를 추출해서 핵심 내용을 정리합니다. **Step 2**에서는 분석 결과를 python-pptx 라이브러리로 PPT 슬라이드에 담습니다. **Step 3**에서는 같은 내용을 이메일 형식으로 변환하되, 받는 사람에 따라 톤을 다르게 생성합니다.

## 지원하는 파일 형식

이 스킬은 단순히 CSV만 읽는 것이 아닙니다. 업무 현장에서 흔히 쓰이는 12종의 파일 형식을 모두 처리합니다.

| 분류 | 확장자 | 처리 방식 |
|------|--------|----------|
| 테이블형 | `.csv` `.xlsx` `.xls` `.tsv` `.json` | 숫자 분석 → 차트 자동 생성 → PPT |
| 문서형 | `.pdf` `.docx` `.doc` | 텍스트 추출 → 핵심 요약 → PPT |
| 프레젠테이션 | `.pptx` `.ppt` | 내용 추출 → 재구성 → PPT |
| 웹/텍스트 | `.html` `.htm` `.md` `.txt` | 텍스트 파싱 → 요약 → PPT |

테이블형 파일을 넣으면 매출 추이 라인 차트, 카테고리별 파이 차트, TOP 5 막대 차트가 자동으로 생성됩니다. 문서형 파일을 넣으면 핵심 문장과 섹션 구조를 분석하여 슬라이드로 정리합니다.

## 이 스킬은 어떤 상황에서 활성화되나요?

앞서 스킬은 "발화 기반"으로 연결된다고 했습니다. 구체적으로 다음과 같은 대화에서 이 스킬이 자동으로 활성화됩니다.

| 이런 말을 하면 | 스킬이 하는 일 |
|---|---|
| "이 파일로 주간 보고 만들어줘" | 전체 파이프라인 실행 |
| "매출 데이터 분석해서 PPT로 만들어줘" | 데이터 분석 + 차트 + PPT |
| "이 PDF 내용을 PPT로 정리해줘" | 텍스트 추출 + 슬라이드 재구성 |
| "이 보고서 요약해서 팀 이메일 써줘" | 문서 요약 + 이메일 생성 |
| "매출 추이 차트 그려줘" | 차트만 생성 |
| "이 워드 파일을 PPT로 바꿔줘" | Word → PPT 변환 |

반대로 "처음부터 글을 써줘"처럼 파일 없이 작성만 요청하거나, 코드 리뷰 같은 개발 작업에서는 이 스킬이 활성화되지 않습니다.

## 만들어 봅시다

### 프로젝트 구조

완성된 스킬의 전체 파일 구조는 다음과 같습니다.

```
doc-automation/
├── SKILL.md                  ← 스킬의 두뇌 (YAML + 지시사항)
├── scripts/
│   ├── generate_report.py    ← 전체 파이프라인 실행
│   ├── analyze_data.py       ← 파일 분석 (12종 형식 지원)
│   ├── create_pptx.py        ← PPT 5장 생성
│   ├── generate_email.py     ← 임원용/팀원용 이메일
│   ├── create_brand_template.py  ← 브랜드 템플릿 생성
│   └── requirements.txt      ← Python 의존성
├── samples/
│   └── weekly_sales.csv      ← 실습용 샘플 데이터
├── templates/email/
│   ├── executive.md          ← 임원용 이메일 톤 가이드
│   └── team.md               ← 팀원용 이메일 톤 가이드
└── README.md
```

핵심 파일은 크게 두 종류입니다. `SKILL.md`는 Claude에게 "언제, 어떻게 이 스킬을 쓸 것인지"를 알려주는 지시서이고, `scripts/` 폴더의 Python 파일들은 실제로 데이터를 분석하고 PPT를 만들어내는 실행 코드입니다.

### Step 1: SKILL.md 작성하기

SKILL.md의 핵심 구성 요소를 살펴보겠습니다.

```yaml
---
name: doc-automation
description: 다양한 형식의 파일(CSV, 엑셀, PDF, Word, PPT, HTML,
  Markdown 등)을 분석하여 주간 보고 PPT와 요약 이메일을 자동
  생성합니다. 보고서 작성, PPT 생성, 데이터 분석, 문서 요약이
  필요할 때 사용하세요.
---
```

YAML 프론트매터 규칙:
- **name**: 소문자와 하이픈(-)만 사용 가능, 64자 이내
- **description**: 200자 이내, Claude가 스킬 활성화 여부를 판단하는 핵심 텍스트

description에는 두 가지를 반드시 포함해야 합니다. 첫째, 이 스킬이 **무엇을 하는지**(파일을 분석하여 PPT와 이메일을 생성). 둘째, **언제 사용하는지**(보고서 작성, PPT 생성, 데이터 분석이 필요할 때). 이 두 정보가 없으면 Claude가 적절한 타이밍에 스킬을 불러오지 못합니다.

프론트매터 아래에는 워크플로우, 지원 파일 형식, 실행 명령어 등을 마크다운으로 상세히 기술합니다. 이 부분이 길수록 Claude가 더 정확하게 작업을 수행합니다.

### Step 2: 분석 엔진 — analyze_data.py

이 스크립트는 파일 확장자를 보고 두 갈래로 분기합니다.

```
입력 파일
   │
   ├─ .csv, .xlsx, .tsv, .json  →  테이블형 분석
   │                                 (pandas → 지표 계산 → matplotlib 차트)
   │
   └─ .pdf, .docx, .pptx,       →  문서형 분석
      .html, .md, .txt               (텍스트 추출 → 섹션/핵심문장 분석)
```

테이블형 파일이 들어오면 pandas로 데이터를 읽고, 합계·평균·TOP 항목 등 주요 지표를 계산한 뒤, matplotlib으로 라인 차트, 파이 차트, 막대 차트를 자동 생성합니다.

문서형 파일이 들어오면 각 형식에 맞는 파서를 사용합니다. PDF는 pdfplumber, Word는 python-docx, HTML은 BeautifulSoup, PPT는 python-pptx로 텍스트를 추출합니다. 추출된 텍스트에서 핵심 문장, 헤딩 구조, 수치 데이터를 분석합니다.

### Step 3: PPT 생성 — create_pptx.py

python-pptx 라이브러리로 5장짜리 슬라이드를 생성합니다. 각 슬라이드의 역할은 다음과 같습니다.

| 슬라이드 | 역할 | 포함 내용 |
|----------|------|----------|
| 1장 - 표지 | 보고서의 얼굴 | 제목, 기간, 작성일 |
| 2장 - 핵심 KPI | 한눈에 보는 숫자 | KPI 카드 3개 + TOP 5 테이블 |
| 3장 - 상세 분석 | 데이터 시각화 | 차트 이미지 삽입 |
| 4장 - 이슈 & 액션 | 문제와 해결 | 주요 이슈 + 액션 아이템 |
| 5장 - 다음 주 계획 | 앞으로의 방향 | 목표 KPI + 주간 일정 |

브랜드 색상, KPI 카드 레이아웃, 헤더 바 등이 코드로 구현되어 있어서 매번 일관된 디자인의 PPT가 나옵니다.

### Step 4: 이메일 생성 — generate_email.py

같은 분석 결과를 두 가지 톤으로 변환합니다.

**임원용**: 핵심 숫자 3개 + TOP 3 상품 + 1~2줄 결론. 3분 안에 읽을 수 있는 분량으로, 하십시오체를 사용합니다.

**팀원용**: 전체 KPI + TOP 5 상품 + 카테고리별 매출 + 구체적인 액션 아이템. 이모지를 활용하여 가독성을 높이고, 담당자 지정까지 포함합니다.

## 스킬 업로드하기

모든 파일이 준비되면 ZIP으로 묶어서 Claude에 업로드합니다. 이때 반드시 지켜야 할 규칙이 있습니다.

**ZIP 패키징 규칙:**

```
doc-automation.zip
└── doc-automation/        ← 반드시 최상위 폴더가 있어야 합니다
    ├── SKILL.md           ← 파일명이 정확히 SKILL.md여야 합니다
    ├── scripts/
    ├── samples/
    └── ...
```

흔한 실수 두 가지:

1. **최상위 폴더 없이 파일을 바로 넣는 경우** — ZIP 안에 `doc-automation/` 폴더가 루트에 있어야 합니다
2. **SKILL.md 파일명이 다른 경우** — `skill.md`, `Skill.md`, `README.md` 등은 인식되지 않습니다

## 내 업무에 맞게 커스텀하기

이 스킬의 진짜 힘은 커스텀에 있습니다. CSV 컬럼명만 바꾸면 어떤 업종에도 적용할 수 있습니다.

### 유튜버: 주간 채널 성과 보고서

```csv
날짜,영상제목,조회수,좋아요,구독자변화
2026-02-16,AI 활용법 총정리,45200,2100,+340
2026-02-17,클로드 실전 가이드,38700,1800,+280
```

"이 데이터로 채널 성과 보고서 만들어줘"라고 하면, 조회수 트렌드 차트, TOP 5 인기 영상 순위, 구독자 성장 그래프가 포함된 PPT가 나옵니다.

### 쇼핑몰 운영자: 주간 매출 보고서

```csv
날짜,상품명,카테고리,매출,판매량,재고,반품수
2026-02-16,오가닉 세럼,스킨케어,2850000,95,420,3
```

일별 매출 추이, 카테고리별 매출 비중 파이 차트, 반품률 분석까지 자동 생성됩니다.

### 프리랜서: 월간 프로젝트 현황

```csv
클라이언트,프로젝트명,진행률,투입시간,마감일
A사,앱 리디자인,75,120,2026-03-15
B사,웹사이트 구축,40,80,2026-04-01
```

클라이언트별 투입시간, 프로젝트 진행률 차트가 포함된 월간 보고서를 만들 수 있습니다.

### PDF/Word 문서 리포맷

이미 작성된 보고서를 새로운 PPT로 재구성하고 싶을 때도 유용합니다. "이 PDF를 발표용 PPT로 만들어줘"라고 하면, 문서에서 텍스트를 추출하고 핵심 내용을 슬라이드 형태로 정리합니다.
